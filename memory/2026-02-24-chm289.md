# CHM-289 계획서 — 채팅 Rich UI 응답

## 무엇을
AI 채팅 응답 안에 UI 컴포넌트(주가 카드, 멀티 종목, 지수 요약)를 인라인 렌더링.
텍스트 응답 + 데이터 카드가 Bloomberg Terminal 스타일로 혼합됨.

## 전체 흐름
1. RAG에서 실시간 시세 데이터를 컨텍스트로 주입 (기존 동일)
2. BASE_INSTRUCTIONS에 컴포넌트 출력 지시 추가
3. AI가 응답 텍스트에 ```component{...}``` 블록 삽입
4. ChatBubble의 ReactMarkdown code 렌더러에서 `language-component` 감지
5. JSON 파싱 → ChatComponentRenderer → 타입별 컴포넌트

## 컴포넌트 타입
- `stock-card`: 단일 종목 (symbol, name, price, change, changePercent, isKR)
- `multi-stock`: 복수 종목 비교 (stocks 배열)
- `index-summary`: 시장 지수 (indices 배열)

## 파일 목록
- `features/chat/components/renderers/stock-card.tsx` (신규)
- `features/chat/components/renderers/multi-stock.tsx` (신규)
- `features/chat/components/renderers/index-summary.tsx` (신규)
- `features/chat/components/chat-component-renderer.tsx` (신규)
- `features/chat/components/chat-bubble.tsx` (수정 — language-component 핸들러)
- `app/api/chat/send/route.ts` (수정 — BASE_INSTRUCTIONS에 컴포넌트 지시 추가)

## 결정 사항
- AI가 RAG 데이터에서 직접 JSON 채움 → 클라이언트 재조회 없음
- 스트리밍 중엔 code block으로 표시 → ``` 닫히는 순간 자동 컴포넌트 전환 (ReactMarkdown 네이티브)
- 데이터 없으면 컴포넌트 블록 생략 (AI 지시에 포함)
